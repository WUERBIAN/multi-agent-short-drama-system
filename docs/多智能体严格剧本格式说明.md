# 多智能体严格剧本格式说明

本系统采用“界面极简，后台多智能体”的结构。

## 调用流程

1. 制片人智能体：完成项目定位、受众、平台风格和商业可行性分析。
2. 编剧策划智能体：完成人物设定、故事主线、人物关系和分集大纲。
3. 编剧团队智能体：按照分集大纲逐集生成完整剧本，每集单独调用一次 API。
4. 导演审核智能体：统一审核全部初稿，提出拍摄可行性、镜头、节奏和成本控制建议。
5. 定稿智能体：根据全部智能体结果整合最终版剧本。

## 严格格式控制方法

系统在编剧团队和定稿智能体的提示词中加入固定的“严格剧本格式规范”，包括：

- 每集必须有本集信息。
- 每集必须有指定数量的场景。
- 每个场景必须有场景编号、内/外、地点、时间、出场人物、场景目标、画面说明、动作、对白和转场。
- 对白必须逐句展开。
- 不允许输出“此处省略”“略”“后续可扩展”。
- 不允许把正式剧本写成剧情梗概。
